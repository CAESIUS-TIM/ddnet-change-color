# TODO 与功能路线图

## 🚀 高优先级

### 1. 颜色预设管理
- **目标**: 支持保存和加载颜色组合预设
- **功能**:
  - 保存当前颜色组合为预设
  - 从预设库加载颜色组合
  - 预设分类管理（游戏、主题、自定义等）
  - 预设预览和描述
- **技术实现**:
  - 扩展 `ColorStore` 支持预设存储
  - 新增预设管理对话框
  - 预设文件格式（JSON 或 YAML）

### 2. 更多颜色格式支持
- **目标**: 支持多种颜色输入格式
- **功能**:
  - RGB 格式支持 (rgb(255, 0, 0))
  - HSL 格式支持 (hsl(0, 100%, 50%))
  - CMYK 格式支持
  - 颜色名称支持 ("red", "blue", "green")
- **技术实现**:
  - 扩展 `is_valid_color` 函数
  - 颜色格式转换工具函数
  - 输入框支持多种格式识别

## 📈 中优先级

### 3. 导入/导出增强
- **目标**: 支持从多种来源导入颜色
- **功能**:
  - 从图片提取颜色主题
  - 从调色板文件导入 (GPL, ASE, ACV)
  - 从 DDNet 配置文件直接导入
  - 导出为图片色板
- **技术实现**:
  - 集成 Pillow 库处理图片
  - 调色板文件解析器
  - 批量导入对话框

### 4. 主题切换
- **目标**: 支持深色/浅色主题切换
- **功能**:
  - 系统主题跟随
  - 手动主题切换
  - 自定义主题颜色
  - 主题预览
- **技术实现**:
  - Qt 样式表动态切换
  - 主题配置存储
  - 高对比度模式支持

### 5. 快捷键自定义增强
- **目标**: 提供更灵活的快捷键配置
- **功能**:
  - 多个绑定键支持
  - 组合快捷键配置
  - 快捷键冲突检测
  - 快捷键预设
- **技术实现**:
  - 扩展 `is_valid_bind_key` 支持组合键
  - 快捷键配置对话框
  - 快捷键冲突检查算法

## 🔧 低优先级

### 6. 批量操作支持
- **目标**: 支持对多个颜色进行批量操作
- **功能**:
  - 批量修改颜色
  - 批量删除颜色
  - 颜色分组管理
  - 操作历史记录和撤销
- **技术实现**:
  - 多选 UI 支持
  - 批量操作命令模式
  - 撤销/重做栈实现

### 7. 颜色对比度检查
- **目标**: 确保生成的颜色配置具有良好的可读性
- **功能**:
  - WCAG 对比度检查
  - 颜色无障碍建议
  - 对比度模拟预览
  - 自动调整建议
- **技术实现**:
  - 对比度计算算法
  - 颜色可读性评估
  - 自动调整算法

### 8. 国际化支持
- **目标**: 支持多语言界面
- **功能**:
  - 英语/中文界面切换
  - 可扩展的语言文件
  - 语言自动检测
- **技术实现**:
  - Qt 国际化框架
  - 翻译文件管理
  - 动态语言切换

## 🤝 贡献指南

欢迎贡献！请参考以下步骤：

1. **选择任务**: 从 TODO 列表中选择一个任务
2. **讨论实现**: 在 Issue 中讨论实现方案
3. **开发实现**: 遵循项目编码规范
4. **测试验证**: 确保测试通过且覆盖率不下降
5. **提交 PR**: 提供清晰的 PR 描述

### 开发规范
- 代码风格遵循 ruff 配置
- 所有新功能需包含测试
- API 变更需更新文档
- 提交信息遵循约定式提交

### 测试要求
- 单元测试覆盖率 ≥ 94%
- 集成测试覆盖主要用户流程
- UI 测试使用 pytest-qt
